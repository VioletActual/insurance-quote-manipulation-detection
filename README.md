# Behavioural Detection of Suspicious Quote Journeys in Motor Insurance

## TL;DR

This project simulates motor insurance **quote journeys** and explores how behavioural analytics can detect suspicious premium-lowering behaviour across repeated quote attempts.

Instead of analysing quotes individually, the project models how customers may repeatedly adjust rating factors (mileage, accidents, experience, age) to reduce premium.

**Key elements**

- Simulated multi-step quote journeys  
- Behavioural feature engineering  
- Rule-based baseline vs machine learning models  
- Random Forest detection model  
- Behaviour shift / drift testing  

**Best model**

Random Forest  
F1 Score: **0.709**  
PR-AUC: **0.811**

The project demonstrates how journey-level behavioural monitoring could support **pricing integrity controls and quote validation workflows**.

## Project Overview

This project explores how behavioural analytics can be used to detect suspicious quote journeys in motor insurance.

Rather than treating each quote as an isolated record, the project simulates repeated quote attempts for the same customer and analyses how details change across the journey.

A "quote journey" refers to the sequence of quotes generated by a customer while experimenting with different inputs before purchasing (or abandoning) a policy.

The focus is on identifying patterns that may indicate deliberate premium-lowering behaviour, such as repeated edits to material rating factors within a short period.

The goal is not to replicate a real insurer system, but to build a realistic prototype demonstrating how behavioural monitoring could support:

- pricing integrity controls  
- quote validation workflows  
- operational referral systems  
- monitoring of quote behaviour patterns  

---

## Why Quote Manipulation Matters

Motor insurance premiums depend on rating factors such as:

- driver age  
- driving experience  
- accident history  
- annual mileage  
- vehicle characteristics  

Customers can sometimes **experiment with different combinations of inputs** across multiple quote attempts to find the lowest premium.

Potential risks include:

- underpriced business  
- inaccurate rating factor data  
- pricing integrity issues  
- operational fraud risk  

This project focuses on **behaviour across repeated quote attempts**, rather than analysing a single quote.

Common behavioural patterns include:

- reducing declared mileage across quotes  
- removing previous accidents  
- increasing declared driving experience  
- adjusting age around rating thresholds  
- repeatedly changing inputs until premium drops

## Dataset

The base dataset is a simplified car insurance premium dataset containing:

- Driver Age  
- Driver Experience  
- Previous Accidents  
- Annual Mileage  
- Car Manufacturing Year  
- Car Age  
- Insurance Premium  

The dataset itself contains **single quotes only** and no quote history.

To study behavioural manipulation, the project extends the dataset by simulating **quote journeys**.

Real insurer pricing engines are significantly more complex and may include **hundreds of rating factors**.  
The premium logic used here is intentionally simplified for simulation purposes.

---

## Quote Journey Simulation

Each customer is assigned a simulated quote journey consisting of multiple quote attempts.

The simulation generates:

- repeated quotes per customer  
- timestamp gaps between quotes  
- edits to rating factors  
- recalculated premium after each edit  
- behavioural overlap between normal and suspicious journeys  

### Rating factors that may change

Across quote attempts customers may modify:

- annual mileage  
- previous accidents  
- driving experience  
- driver age  

Age edits are intentionally small and realistic, especially around known pricing boundaries:

- **24 → 25**  
- **39 → 40**  
- **65 → 64**  

### Age / Experience Consistency

The project includes checks for implausible combinations such as unusually high driving experience relative to age.

### Hidden Behavioural Intent

To prevent labels being directly derived from observable features, the simulation includes a **hidden behavioural intent signal**.

This introduces additional randomness and overlap between normal and suspicious journeys, making the modelling task more realistic.

## Feature Engineering

The project constructs **journey-level behavioural features** from the quote sequence.

Examples include:

- number of quote attempts  
- first-to-lowest premium drop %  
- cumulative premium reduction  
- largest premium drop  
- mileage decrease count  
- accident removal count  
- experience increase count  
- age edits and age-band crossings  
- consistency violations  
- number of material edits  
- suspicious step ratio  
- number of unique fields edited  
- time between quotes  
- quote velocity score  

These features describe **how a customer behaves across the quote journey**, rather than only describing the final quote.

## Detection Approaches

Several approaches are used to identify suspicious journeys.

### 1. Heuristic baseline

A simple rule-based screen designed to capture obvious cases such as:

- many quote attempts  
- large premium reductions  
- multiple material edits  
- suspicious behaviour in short time windows  

This represents how insurers often implement **initial validation rules**.

### 2. Supervised machine learning

Models trained on behavioural features:

- Logistic Regression  
- Random Forest  
- Histogram Gradient Boosting  

### 3. Unsupervised anomaly detection

Isolation Forest is used as a comparison benchmark.

Isolation Forest identifies **unusual behavioural patterns without using the suspicious label**, providing an unsupervised detection baseline.

## Model Results

### Best performing model

The strongest model in the final Version 2 setup is **Random Forest**.

| Model | Precision | Recall | F1 | PR-AUC |
|------|-----------|--------|----|--------|
| Logistic Regression | < RF | < RF | < RF | < RF |
| **Random Forest** | **0.824** | **0.622** | **0.709** | **0.811** |
| Hist Gradient Boosting | < RF | < RF | < RF | < RF |

### Heuristic baseline

The heuristic rule screen behaves as a **high-precision filter**.

- Precision: **0.800**  
- Recall: **0.178**  
- F1: **0.291**

This is expected: rules capture only the most obvious cases, while machine learning detects more subtle behavioural patterns.

### Class balance

Approximately **4.5% of journeys** are labelled suspicious in the final simulation.

This provides a realistic imbalance while still allowing meaningful evaluation.

---

## Behaviour Shift Test

Customer behaviour can change over time. To test robustness, the project includes a **shifted simulation scenario**.

This evaluates how the trained model performs when behavioural patterns evolve.

### Shifted-behaviour results

- Precision: **0.704**  
- Recall: **0.613**  
- F1: **0.655**

Performance degrades slightly but remains usable, which is expected under behavioural drift.

## Example Suspicious Journey Patterns

Suspicious journeys typically include patterns such as:

- multiple quote attempts in a short time window  
- repeated premium-lowering edits  
- several material rating factors changing  
- age-band threshold crossing  
- implausible age/experience combinations  
- large first-to-lowest premium reductions  

Example journey pattern:

- Quotes: 5–6 attempts  
- Premium reduction: 20%+  
- Edits: mileage, accidents, experience  
- Short time gaps between quotes  

These behavioural indicators **do not prove fraud**, but may justify additional validation.

## Candidate Behavioural Indicators

The strongest behavioural signals observed include:

- material edit frequency  
- suspicious edit ratios  
- quote velocity  
- cumulative premium reduction  
- age band crossing behaviour  
- consistency violations  

These signals could support:

- pricing integrity monitoring  
- quote validation workflows  
- operational referral systems  

They are **not intended as direct rating factors** within pricing models.

## Limitations

This project is a realistic prototype but has several limitations.

### Simulated behaviour

Quote journeys are simulated rather than derived from real insurer quote streams.

### Simplified pricing function

The premium calculation used here is intentionally simplified.

### Limited rating factors

The dataset does not include variables such as postcode, occupation, no-claims discount, or additional drivers.

### Prototype scope

The project demonstrates behavioural analytics concepts rather than a production deployment.
